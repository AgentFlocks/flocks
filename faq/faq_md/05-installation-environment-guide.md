# Flocks 安装环境准备指南

## 摘要
本文整理安装 Flocks 之前需要确认的系统依赖、版本要求和环境差异。根据 README，核心依赖包括 `uv`、`Node.js` 与 `npm 22+`、`agent-browser`，可选依赖为 `bun`。安装前把这些基础环境确认好，可以显著降低后续脚本失败和 WebUI 构建失败的概率。

## 适用场景
- 还没开始安装，想先确认环境是否满足要求
- 一键安装过程中经常失败，怀疑依赖不完整
- 需要提前评估本机、服务器、虚拟机或容器是否适合部署 Flocks

## 依赖项
安装前重点确认：

- `uv`
- `Node.js`
- `npm 22.+`
- `agent-browser`
- `bun`（可选，仅 TUI 安装相关）

README 还特别说明：如果自动安装 `npm` 失败，需要手动安装 `npm 22+`。

## 各依赖分别影响什么
### `uv`
主要影响 Python 环境和后端依赖安装，是后端运行的基础组件。

### `Node.js` 与 `npm`
主要影响 WebUI 依赖安装和构建。如果这部分异常，常见表现是安装脚本卡住、WebUI 构建失败、前端无法正常启动。

### `agent-browser`
主要影响浏览器自动化能力。如果浏览器相关依赖安装失败，往往不会只影响“打开网页”这一个功能，还可能影响部分接入场景。

### `bun`
属于可选能力，主要和 TUI 安装有关，不是 WebUI 正常运行的强制依赖。

## 不同部署环境的准备重点
### 本机安装
最适合完整能力验证，尤其适合需要浏览器自动化、网页登录和可视化排障的场景。

### 云服务器安装
重点不是桌面环境，而是后端和 WebUI 是否能正常安装，以及浏览器相关能力是否只能使用无头模式。

### Docker 安装
更适合快速起服务和隔离环境，但需要额外确认端口映射、目录挂载以及浏览器交互限制。


## 推荐的安装前检查清单
- 已确认系统可以安装或访问 `uv`
- 已确认 `npm` 版本满足 `22+`
- 已确认是否需要浏览器自动化能力
- 已确认磁盘空间是否足够
- 已确认自己最终是走本机安装、源码安装还是 Docker 安装

## 常见误区
### 认为浏览器依赖失败只影响浏览器
群聊中的实际经验表明，浏览器依赖失败有时会导致整个安装流程没有真正完成。

### 认为只要有 Python 就够了
Flocks 不只是后端服务，还包含 WebUI，因此 Node.js 和 npm 同样关键。
