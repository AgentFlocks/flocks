# Flocks MCP 与工具调用指南

## 摘要
本文说明 Flocks 中 MCP 配置、认证、测试 JSON、工具调用结果保存等常见问题。实践中最重要的是先区分“服务地址配置是否正确”“认证是否完整”“测试方式是否符合工具要求”，而不是一开始就把问题归因到模型本身。

## 适用场景
- 想配置 MCP 服务
- 页面上只看到服务地址，不知道怎么认证
- MCP 测试时 JSON 报错
- 工具调用成功但结果没有按预期保存

## MCP 配置的基本思路
配置 MCP 时，至少要确认三件事：
- 服务地址是否正确
- 认证信息是否完整
- 当前测试方式是否符合该 MCP 的预期输入

## 页面上只有服务地址怎么办
群聊中有用户提到“页面上只看到服务地址，没有认证入口”。这类情况通常说明：
- 你当前看到的是最基础的连接配置
- 认证过程可能依赖该服务的特定接入方式
- 需要参考对应 MCP 的接入示例或文档

## MCP 测试 JSON 报错
这类问题常见于：
- JSON 格式本身错误
- 输入内容不符合测试界面预期
- 应该在对话中调用，而不是直接在测试框里粘贴

群聊中就出现过类似判断：某些 JSON 更适合让 Flocks 在对话中实际调用，而不是直接丢进测试框。

## 工具调用成功但结果没有自动保存
这类现象在 Docker 或特定工作目录场景中更容易出现。排查方向通常是：
- 结果是否真的已生成，只是你没看对目录
- 当前工具调用是否本来就不自动落盘
- 容器挂载目录是否正确

## 推荐排障顺序
1. 确认服务地址
2. 确认认证方式
3. 确认输入格式
4. 确认结果查看位置
