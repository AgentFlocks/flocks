# Flocks Skill 与系统接入指南

## 摘要
本文说明 Flocks 在接入 TDP、OneSec、SIP、XDR 等系统时，什么时候优先使用 API，什么时候需要浏览器页面接入，以及如何通过更清晰的提问方式提高 Skill 的命中率。社区经验表明，模型能力越一般，提示词就越需要明确写出目标系统、Skill 名称和工具名称。

## 适用场景
- 想接入安全平台或外部系统
- 已安装 Skill，但 Agent 没按预期调用
- 不知道该走 API、Skill 还是网页浏览方式

## 优先级原则
### 第一优先：API 接入
如果目标系统有稳定 API，通常优先选 API。原因是：
- 稳定性更高
- 可重复性更强
- 更适合后续自动化和定时分析

### 第二优先：Skill 规范化调用
如果系统已有现成 Skill，提问时应尽量显式写出：
- 使用哪个 Skill
- 使用哪个工具
- 目标任务是什么

### 第三优先：浏览器页面接入
如果没有 API，或者 API 不足以覆盖目标数据，再考虑让 Flocks 通过网页读取数据。

## 为什么“问法”很重要
群聊里多次提到，对于能力一般的模型，模糊提问会导致：
- 不会调用正确工具
- 开始走错误路线
- 明明有 Skill，却去尝试网页模拟操作

更稳妥的写法是：
- 明确目标系统
- 明确使用某个 Skill 或工具
- 明确要查询、筛选或分析的结果

## Skill 的安装与落盘
群聊中提到，Skill 可以通过安装方式生成，也可以放到本地插件目录，例如：

```text
~/.flocks/plugins/skills
```

这适合团队内部积累成功经验后沉淀为可复用能力。

## API 接入和页面接入各适合什么
### API 接入适合
- 告警查询
- 数据获取
- 定时分析
- 稳定化自动流程

### 页面接入适合
- 没有 API 的系统
- 某些必须登录后才能查看的内容
- 临时验证和小规模场景
