# Flocks 升级失败排查指南

## 摘要
本文用于处理升级过程中最常见的失败场景，包括升级长时间卡住、页面升级失败、Windows 升级失败、升级后无法启动以及旧进程未退出导致文件无法替换。遇到升级问题时，最稳妥的思路通常是先 `flocks stop`，再切换到源码手动升级。

## 适用场景
- 点击升级后失败
- 升级长时间无响应
- 升级完成后无法启动
- Windows 升级频繁失败

## 常见失败类型
### 升级卡很久
群聊中多次出现“卡了二三十分钟”“卡了一小时”的反馈。判断方法是：
- 如果仍有新输出，可能只是慢
- 如果长时间没有任何变化，多半已经失败

### 页面升级失败
通常说明：
- 当前版本本身存在升级兼容问题
- 本地文件替换失败
- 前端或后端进程没有完全退出

### Windows 升级失败
这是群聊里最典型的高频问题之一。社区经验普遍认为，Windows 更适合手动升级，而不是依赖页面一键升级。

## 推荐排障步骤
1. 执行：

```bash
flocks stop
```

2. 确认旧进程已经退出
3. 拉取最新代码或覆盖源码
4. 重新执行安装脚本
5. 再执行：

```bash
flocks restart
```

## 为什么升级会因为旧进程失败
群聊中多次提到，升级失败的根本原因之一是旧的后端进程没有退出，导致文件正在被占用，新的文件无法替换。

这在 Windows 环境中尤其明显。

## 社区中常见的手动升级思路
```bash
flocks stop
git pull
./scripts/install.sh
flocks restart
```

如果是 Windows：
- 使用管理员 PowerShell
- 执行 `install.ps1`

## 是否需要先卸载
大多数情况下不需要先卸载。更常见的做法是：
- 先停服务
- 再重新执行安装脚本覆盖安装

## 什么时候直接放弃页面升级
- 你在 Windows 上
- 当前版本社区中已知升级有问题
- 你已经连续多次页面升级失败
