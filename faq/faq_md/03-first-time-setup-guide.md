# Flocks 首次配置指南

## 摘要
本文面向已经成功安装并打开 WebUI 的用户，说明首次进入 Flocks 后应该完成的关键配置步骤。核心顺序是：添加供应商、添加模型、在模型清单中测试模型、设置默认模型，然后再开始使用 Agent 或其他接入能力。

## 适用场景
- WebUI 已能打开，但不知道先配置什么
- 已经添加了模型，首页仍提示未完成引导
- Agent 无法正常执行，怀疑默认模型没有设置好

## 首次配置的标准顺序
### 1. 进入模型管理或新手引导
首次进入 WebUI 后，优先完成模型相关配置，而不是直接开始对话或集成外部系统。

### 2. 添加模型供应商
如果使用官方支持的模型服务，直接选择对应供应商。

如果使用自建模型、网关、兼容 OpenAI API 的服务，优先选择 `OpenAI Compatible`。

### 3. 添加具体模型
填写：
- API 地址或 Base URL
- API Key
- 模型名称

### 4. 在模型清单中测试连接
群聊中的高频经验是：仅保存模型还不够，很多问题会在模型清单的测试步骤里提前暴露出来。

### 5. 设置默认模型
这是首次配置最容易漏掉的一步。即使你已经添加了模型，如果没有设置默认模型，首页仍可能显示未完成配置，Agent 也可能无法正常工作。

## 默认模型与已添加模型的区别
- 已添加模型：只是说明模型信息被保存到了系统中
- 默认模型：决定 Agent 和首页引导默认使用哪一个模型

很多“模型明明加了但仍不能用”的问题，本质上都是默认模型没有设置。

## 推荐检查清单
- 已成功添加至少一个模型供应商
- 已保存至少一个可用模型
- 已在模型清单中完成测试
- 已设置默认模型
- 首页引导状态已从未完成变成正常

## 高频出现的问题
- 模型已经配置了，但首页仍然是红色提示
- 本地模型已经可用，但系统还在提示免费额度问题
- 用户添加了模型，却没有到模型清单单独测试
- 第三方模型服务没有先建 `OpenAI Compatible` provider，导致后续流程卡住

## 常见错误与排查
### 已经添加了模型，为什么首页还提示未完成配置？
优先检查是否已经设置默认模型。

### 为什么保存模型后，Agent 还是不能正常使用？
优先检查三件事：
1. 模型是否测试通过
2. 是否设置为默认模型
3. Base URL 和模型名称是否真实可用

### 配置看起来没问题，但对话仍异常
这时问题多半已经从“首次配置”转为“模型报错排查”，应查看模型测试结果和后端日志。
