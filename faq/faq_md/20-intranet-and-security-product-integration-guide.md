# Flocks 内网与安全产品接入指南

## 摘要
本文面向需要在内网环境中使用 Flocks、接入本地安全平台或分析局域网告警数据的场景，说明 Flocks 是否必须联网、在纯内网中还能做什么，以及接入安全产品时应优先选择 API、日志推送还是网页登录。总体建议是：有 API 优先 API；没有 API 再考虑浏览器读取；定时分析类需求更适合标准化数据接入。

## 适用场景
- 想把 Flocks 部署在纯内网或受限网络
- 想接入 SIP、XDR、AES、hfish 等安全平台
- 想对本地告警做定时分析或汇总报告

## Flocks 是否必须联网
群聊中的共识是：
- Flocks 并不是硬性要求必须联网
- 但如果你使用的是外部大模型服务，那模型本身仍需要网络
- 如果你有本地部署的大模型，也可以在隔离互联网的环境中使用

## 内网环境中哪些能力最现实
更现实、可持续的做法通常是：
- 使用本地或私有模型
- 通过 API 接入本地安全平台
- 通过日志、消息队列或中转脚本向 Flocks 提供数据

## 安全产品接入的优先顺序
### 第一优先：官方或私有 API
如果平台有 API，这是最适合长期使用的方式。

### 第二优先：日志推送或中间脚本
如果没有完整 API，但可以输出日志、消息或结构化数据，可以用中转脚本对接。

### 第三优先：网页登录取数
只有在完全没有 API 或标准输出方式时，才推荐走浏览器页面读取路径。

## 定时分析场景怎么做
如果你的目标是：
- 每天固定时间汇总告警
- 自动生成分析报告
- 按某种规则筛选本地事件

最推荐的方式仍是：
- 先把数据标准化接入
- 再让 Flocks 负责分析与总结

而不是每次都依赖人工打开页面。

## 适用边界也要说明
群聊中有一个很重要的共识：Flocks 更擅长对单条线索、单次事件或结构化任务做深度分析，而不是直接替代传统检测系统去做大规模实时检测。
