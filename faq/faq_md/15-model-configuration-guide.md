# Flocks 模型配置指南

## 摘要
本文说明 Flocks 中模型供应商、模型实例、模型测试和默认模型之间的关系，并给出最稳妥的配置顺序。核心顺序是：先建供应商，再建模型，再在模型清单里测试，最后设置默认模型。很多“模型已经配了但系统仍异常”的问题，都出在最后两步。

## 适用场景
- 第一次给 Flocks 配置模型
- 模型已经添加，但首页仍提示未完成
- 不清楚默认模型和模型清单之间的关系

## 配置顺序
### 1. 添加模型供应商
优先区分两种情况：
- 官方已支持的模型厂商
- 自己的兼容服务或第三方网关

### 2. 添加具体模型
填写：
- Base URL
- API Key
- 模型名

### 3. 到模型清单中测试
这一步很关键。群聊中的经验说明，仅保存配置不等于模型已经真正可用。

### 4. 设置默认模型
这是控制 Agent 和首页引导状态的关键步骤。

## 为什么默认模型这么重要
- 已添加模型：只是保存了配置
- 默认模型：决定系统当前实际优先使用哪个模型

如果没有设置默认模型，常见现象是：
- 首页仍显示未完成配置
- Agent 行为异常
- 系统似乎还在走原来的模型或免费额度

## 推荐配置检查清单
- 供应商已创建
- 模型已保存
- 模型测试通过
- 默认模型已设置

## 常见错误与排查
### 为什么填了参数后不显示模型？
优先检查 Base URL 是否真实可访问，以及当前接入方式是不是兼容 OpenAI API 的模式。

### 为什么模型测试过了，对话还是报错？
模型配置本身可能没问题，但对话阶段还可能受到模型负载、上下文、响应格式和平台限制影响。
